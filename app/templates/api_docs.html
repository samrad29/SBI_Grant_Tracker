SBI Grant Tracker API docs

Reference for frontend integration. All endpoints below are currently GET and return JSON.

Use URL query parameters exactly as documented. For write actions (bookmark/checklist updates), requests still use query params even though they mutate data.

← Mission Control

Core data endpoints (`api.py`)

GET/api/opportunities

Lists opportunities. Supports either tag search, text search, or default top 50.

Query params
  • tags (optional): comma-separated tags, example education,housing
  • q (optional): free-text search on title/agency, example tribal health
Response shape
  • No params or q: array of basic grant objects
  • With tags: array of scored objects including total_score and tag_scores
Example response (`tags=education,housing`)
[
  {
    "opportunity_id": "336489",
    "title": "Community Learning Grant",
    "agency": "Department of Education",
    "status": "posted",
    "total_score": 18.0,
    "tag_scores": [
      { "tag": "education", "tag_score": 10.0 },
      { "tag": "housing", "tag_score": 8.0 }
    ]
  }
]
GET/api/opportunities/<opportunity_id>

Returns full grant details from the grants table for one opportunity.

Path params
  • opportunity_id (required): opportunity identifier string
Example response (200)
{
  "opportunity_id": "336489",
  "number": "ED-GRANT-2026-01",
  "title": "Community Learning Grant",
  "agency": "Department of Education",
  "status": "posted",
  "posted_date": "2026-03-10",
  "close_date": "2026-06-15",
  "description": "Funding for local education initiatives.",
  "updated_at": "2026-04-12T03:41:22"
}
Not found
{}
GET/api/alerts

Returns the 50 most recent alert records from grant_alerts.

Query params
  • None
Example response
[
  {
    "id": 421,
    "opportunity_id": "336489",
    "detected_at": "2026-04-13T10:07:00",
    "alert_type": "deadline_changed",
    "field": "close_date",
    "old_value": "2026-06-01",
    "new_value": "2026-06-15"
  }
]

User activity endpoints (`user_activity.py`)

GET/api/user_activity/update_user_grant_status

Creates or updates a user's status for a grant in user_grant_activity.

Query params
  • user_id (required): user identifier string
  • opportunity_id (required): grant id
  • status (required): arbitrary status string (for example saved, interested, submitted)
Example response
{ "message": "Grant marked successfully" }
GET/api/user_activity/bookmark_grant

Bookmarks a grant for a user (is_bookmarked = TRUE).

Query params
  • user_id (required)
  • opportunity_id (required)
Example response
{ "message": "Grant bookmarked successfully" }
GET/api/user_activity/unbookmark_grant

Unbookmarks a grant for a user (is_bookmarked = FALSE).

Query params
  • user_id (required)
  • opportunity_id (required)
Example response
{ "message": "Grant unbookmarked successfully" }
GET/api/user_activity/get_bookmarked_grants

Returns all bookmarked grants for a user from user_grant_activity, newest first.

Query params
  • user_id (required)
Example response
[
  {
    "user_id": "demo-user",
    "opportunity_id": "336489",
    "status": "saved",
    "is_bookmarked": true,
    "created_at": "2026-04-12T18:31:49"
  }
]
GET/api/user_activity/get_user_alerts

Returns up to 50 alerts for bookmarked grants for the specified user.

Query params
  • user_id (required)
Example response
[
  {
    "user_id": "demo-user",
    "opportunity_id": "336489",
    "status": "saved",
    "created_at": "2026-04-12T18:31:49",
    "alert_type": "deadline_changed",
    "field": "close_date",
    "old_value": "2026-06-01",
    "new_value": "2026-06-15",
    "detected_at": "2026-04-13T10:07:00"
  }
]
GET/api/user_activity/get_checklist_items

Returns checklist items for one user + one opportunity, ordered by item_id.

Query params
  • user_id (required)
  • opportunity_id (required)
Example response
[
  {
    "item_id": 12,
    "user_id": "demo-user",
    "opportunity_id": "336489",
    "item_name": "Draft narrative",
    "is_completed": false
  }
]
GET/api/user_activity/update_checklist_item

Updates is_completed for one checklist item.

Query params
  • user_id (required)
  • opportunity_id (required)
  • item_id (required)
  • is_completed (required): accepts true/false, 1/0, yes/no, t/on
Example response
{ "message": "Checklist item updated successfully" }
GET/api/user_activity/add_checklist_item

Adds a new checklist item for a user and grant.

Query params
  • user_id (required)
  • opportunity_id (required)
  • item_name (required): checklist text
Example response
{ "message": "Checklist item added successfully" }
Error format: endpoints generally return HTTP 500 with JSON like { "message": "Error ...: <details>" } when exceptions occur.
